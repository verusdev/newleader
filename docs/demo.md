# Демо-сценарий

## Цель

Этот документ нужен для демонстрации проекта заказчику или стейкхолдерам.

## Запуск

```bash
php artisan serve
```

Опционально:

```bash
npm run dev
```

## Рекомендуемая последовательность показа

### 1. Главный SaaS-лендинг

Открыть:

`http://127.0.0.1:8000/`

Что показать:

- это центральная точка входа в продукт
- здесь начинается сценарий подписки
- отсюда новый ведущий может зайти в систему

### 2. Админка tenant-ов

Открыть:

`http://127.0.0.1:8000/admin/tenants`

Что показать:

- администратор видит всех ведущих / tenant-ов
- у каждого tenant-а есть CRM
- у каждого tenant-а есть публичный рекламный лендинг
- дизайн лендинга можно менять отдельно для каждого ведущего

### 3. Карточка tenant-а

Открыть:

`http://127.0.0.1:8000/admin/tenants/96786bbd-3a80-4f1f-95c0-c39947c15b46`

Что показать:

- метаданные tenant-а хранятся централизованно
- администратор может открыть CRM
- администратор может открыть публичный лендинг
- шаблон лендинга меняется прямо из карточки

### 4. Публичный лендинг ведущего

Открыть:

`http://127.0.0.1:8000/hosts/96786bbd-3a80-4f1f-95c0-c39947c15b46`

Что показать:

- это рекламная страница ведущего
- у неё есть несколько визуальных шаблонов
- часть данных может подтягиваться из tenant CRM

### 5. Tenant CRM

Открыть:

`http://127.0.0.1:8000/tenant/96786bbd-3a80-4f1f-95c0-c39947c15b46`

Что показать:

- это рабочее пространство ведущего
- здесь находятся мероприятия, клиенты, задачи, гости и бюджет
- данные tenant-а изолированы в отдельной базе

## Демонстрация переключения шаблонов

Чтобы показать вариативность дизайна:

1. Открыть карточку tenant-а в админке
2. Изменить `landing_template`
3. Сохранить
4. Обновить `/hosts/{tenant-id}`

Доступные стили:

- `classic`
- `editorial`
- `neon`

## Замечания для демо

- Если tenant-база ещё не создана, публичный лендинг всё равно открывается с пустой статистикой.
- Tenant SQLite-файлы являются локальными данными окружения и не предназначены для хранения в репозитории.
- Текущая реализация использует path-based tenancy, что упрощает локальное демо.
