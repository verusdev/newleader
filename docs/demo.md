# Демо-сценарий

## Цель

Этот документ нужен для демонстрации проекта заказчику или стейкхолдерам.

## Запуск

```bash
php artisan serve
```

Опционально:

```bash
npm run dev
```

Если нужно заново наполнить demo tenant-ы:

```bash
php scripts/refresh_demo_data.php
```

## Рекомендуемая последовательность показа

### 1. Главный SaaS-лендинг

Открыть:

`http://127.0.0.1:8000/`

Что показать:

- это центральная точка входа в продукт;
- здесь начинается сценарий продажи подписки;
- отсюда ведущий попадает в экосистему продукта.

### 2. Админка tenant-ов

Открыть:

`http://127.0.0.1:8000/admin/tenants`

Что показать:

- все tenant-ы собраны централизованно;
- у каждого tenant-а есть CRM;
- у каждого tenant-а есть публичный host landing;
- шаблон host landing можно менять отдельно.

### 3. Публичный лендинг ведущего

Открыть любой из:

- `http://127.0.0.1:8000/hosts/906a4463-2114-4c3d-aa8d-d198c7727be9`
- `http://127.0.0.1:8000/hosts/96786bbd-3a80-4f1f-95c0-c39947c15b46`

Что показать:

- это рекламная страница ведущего;
- у страницы есть разные шаблоны дизайна;
- часть данных подтягивается из tenant CRM.

### 4. CRM ведущего

Открыть, например:

`http://127.0.0.1:8000/tenant/96786bbd-3a80-4f1f-95c0-c39947c15b46`

Что показать:

- рабочее пространство ведущего;
- лиды и клиенты;
- мероприятия;
- календарь мероприятий;
- задачи, бюджет, подрядчики, гости.

### 5. Воронка лида

В CRM открыть раздел клиентов.

Что показать:

- контакт сначала создаётся как лид;
- информация о мероприятии фиксируется уже на этом этапе;
- есть таймлайн сделки;
- при завершении этапа `Заключение договора` лид становится клиентом.

### 6. Календарь мероприятий

Открыть страницу календаря мероприятий в tenant CRM.

Что показать:

- все мероприятия видны в одном календаре;
- есть просмотр по месяцам, неделям и списком;
- по событию можно перейти в карточку мероприятия.

### 7. Карточка мероприятия

Открыть любое мероприятие.

Что показать:

- вся операционная информация по событию;
- список гостей;
- задачи;
- бюджет;
- ссылка на публичный invitation landing.

### 8. Invitation landing и RSVP

Из карточки мероприятия открыть публичное приглашение.

Что показать:

- у каждого события есть отдельный invitation landing;
- можно использовать общий URL для массовой отправки;
- у гостей есть персональные ссылки;
- RSVP заполняется прямо на публичной странице;
- ответ попадает обратно в CRM.

### 9. Список гостей

Открыть раздел гостей внутри мероприятия.

Что показать:

- у каждого гостя есть RSVP-статус;
- видно, кто подтвердил участие, кто отказался, а кто ещё не ответил;
- для каждого гостя доступна персональная invitation ссылка.

## Что заполнено в demo-данных

У каждого demo tenant-а уже есть:

- несколько лидов и клиентов;
- несколько мероприятий с разными статусами;
- гости с русскими именами и RSVP-ответами;
- задачи;
- бюджетные статьи;
- подрядчики.

Это сделано специально для показа:

- CRM выглядела живой;
- календарь был заполнен;
- invitation landing и RSVP можно было сразу демонстрировать.

## Замечания для demo

- В локальном demo проще использовать path-based tenancy.
- Tenant SQLite-файлы являются данными окружения, а не частью исходников.
- Если нужно сбросить и заново наполнить demo-сценарий, используйте [../scripts/refresh_demo_data.php](../scripts/refresh_demo_data.php).
